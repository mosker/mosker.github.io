前端构建工具gulp入门教程

发表于 2015-12-25   |   分类于 js   |  

一、gulp优点

gulp 和 grunt 都是基于Node.js创建的前端构建工具，目前这两个前端构建工具是最流行也是最好用的。 Gulp 作为全新的构建工具，相比于 Grunt 的频繁的 IO 操作，Gulp 的流操作，能更快地完成构建。 另外比起 Grunt 不仅配置简单而且更容易阅读和维护，所以学习 gulp 是前端开发人员的一个绝佳选择。

二、安装gulp

gulp 是基于Node.js，所以在所用这个工具之前需要安装node，本教程是在WIN7环境下。

首先我们要全局安装 gulp ：

npm install gulp -g

运行命令后将gulp安装到全域环境下，让你可以存取gulp的CLI。然后需要在你本地端的项目目录进行安装，进入你的项目根目录，执行下列指令（确保目录有package.json文件）：

npm install gulp --save-dev

运行指令后将gulp安装到项目根目录的专案内，并记录package.json文件devDependencies中，另外 gulp 模块包下载到node_modules文件夹中。

三、使用gulp

Gulp的任务都是以插件的形式存在，本次示例以 gulp-jshint、gulp-concat、gulp-uglify、gulp-rename 为例，展示Gulp的常规用法（更多插件及用法参考官方API）。

1、安装本示例所需插件，运行如下命令：

npm install gulp-jshint gulp-concat gulp-uglify gulp-rename

运行完成后可以看到所需插件已安装，如图：

2、接着我们在根目录下要创建gulpfile.js文件。

3、打开gulpfile.js文件并加载插件：

var gulp = require('gulp'),
    jshint = require('gulp-jshint'),//语法检查
    concat = require('gulp-concat'),//合并文件
    uglify = require('gulp-uglify'),//压缩代码
    rename = require('gulp-rename');//重命名

4、建立任务：

jshint任务：

// 语法检查
gulp.task('jshint', function () {
    return gulp.src('public/javascripts/*.js')
    .pipe(jshint())
    .pipe(jshint.reporter('default'));
    });

minify任务：

// 合并文件之后压缩代码
gulp.task('minify', function (){
    return gulp.src('public/javascripts/*.js')
    .pipe(concat('all.js'))
    .pipe(gulp.dest('public/javascripts/dist'))
    .pipe(uglify())
    .pipe(rename('all.min.js'))
    .pipe(gulp.dest('public/javascripts/dist'));
    });

watch任务：

// 监视文件的变化
gulp.task('watch', function () {
    gulp.watch('public/javascripts/*.js', ['jshint', 'minify']);
    });

预设任务：

// 注册缺省任务
gulp.task('default', ['jshint', 'minify', 'watch']);

gulp default
gulp watch
gulp minify
gulp jshint

四、总结

这个示例只是 gulp 的几个简单的使用方法，gulp 还有很多很强大的插件和功能，想了解的更深入可以访问gulp中文官网。祝大家生活工作愉快！
